## Version 12.0.0 (2025-11-19)

### Breaking changes

- `IqmServerClient` was removed from the `iqm-station-control-client` package.
  Use the new REST-based `_IQMServerClient` in the `iqm-client` package.

### Features

- Add new circuit and job related models to station control interface,
  used in IQM client when serializing/deserializing data in IQMServer communication
- Deprecate `observation_ids` in `ObservationSetWithObservations`, use `observations` instead.
  This is to unify the data with IQMServer.
- `StationControlClient` now accepts the `token` and `tokens_file` parameters,
  and can use the `IQM_TOKEN` and `IQM_TOKENS_FILE` environment variables
  to get the authentication token.
- In case major versions differ, log warning instead of raising an error in the python package
  version compatibility check when connecting to station-control server with
  `iqm.station_control.client.station_control.StationControlClient`.
- `StationControlClient` now accepts the `token` and `tokens_file` parameters,
  and can use the `IQM_TOKEN` and `IQM_TOKENS_FILE` environment variables
  to get the authentication token.
- In case major versions differ, log warning instead of raising an error in the python package
  version compatibility check when connecting to station-control server with
  `iqm.station_control.client.station_control.StationControlClient`.

### Bug fixes

- Restrict RunDefinition fields to fix issues with serializing None and roundtrip resulting in empty dictionary.
- Fix python package compatibility version check when connecting to station-control server.

## Version 9.0.0 (2025-06-13)

### Features

- Reintroduce `.StationControlInterface` and make both `.StationControlClient` and `.IQMServerClient` inherit it
  to implement the same interface.
- Add new methods to `StationControlInterface`, implemented by `StationControlClient`. `IQMServerClient` doesn't
  implement these new methods yet, but raises `NotImplementedError` instead.

  - `get_default_calibration_set`
  - `get_default_calibration_set_observations`
  - `get_dynamic_quantum_architecture`
  - `get_default_dynamic_quantum_architecture`
  - `get_static_quantum_architecture`

- Support also string UUIDs in `StationControlClient` methods, instead of only `UUID` objects. This always worked in
  practice, since UUIDs were instantly serialized to strings anyway. However, it gave type warnings for the users
  when string given. No functional changes, but no more type warnings.

### Breaking changes

- Remove `StationControlClient.init`, `StationControlClient.get_calibration_set_values` and
  `StationControlClient.get_latest_calibration_set_id` methods. Those aren't REST API related
  and needed only by Pulla for now, and they don't belong to `StationControlInterface` since there is already
  `StationControlInterface.query_observation_sets`,
  `StationControlInterface.get_observation_set` and
  `StationControlInterface.get_observation_set_observations`
  which can be used for the same purpose. `IQMServerClient` still has the original methods
  and it should be later refactored to implement the same interface as `StationControlClient`.

### Breaking changes

- This is only a bug fix MR, however it's technically a breaking change for station-control since the fix
  required us to change how list objects are serialized/deserialized and thus, station-control need to use
  different syntax for that from now on. exa-repo or other clients shouldn't be affected, so from their perspective
  this should be considered a minor release.

### Bug fixes

- Fix `NaN` and `Inf` serialization for float values, serializing them to `NaN` and `Inf` instead of `None`.
  This affected only when serializing list of objects at once (for example when saving multiple observations),
  endpoints dealing with single object were working as expected.

### Features

- Add optional `wait_task_completion` boolean parameter (default `True`) to station control client's `run()`.

  receives the initial response. This feature can be used to implement async-like workflows which is not blocked
  by the task execution.
